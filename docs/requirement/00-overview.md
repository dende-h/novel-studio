## 1. これは何か

**ローカルファーストの創作ツール（小説を書く道具）**。書いた小説を複数のターゲットへ「持ち出せる」。

- 主役 = 書く道具（ローカルファースト）
- 公開は**複数ターゲットの一つ**:
  1. 自前プラットフォーム（シームレス連携・**P1+**）
  2. EPUB / 電子書籍 出力
  3. 外部サイト（小説家になろう / カクヨム 等）への記法書き出し
- 読書 / 発見（本屋ぶらぶら・棚＋シャッフル） / 支援 は「**公開プラットフォーム**」コンポーネント（P1+）

## 2. なぜこの形か（旧版からの転換）

旧版は「一体型の投稿プラットフォーム」で、未公開原稿もサーバ前提だった。転換の理由:

- **未公開小説の漏洩防止** — 書く道具はローカル保存を既定にし、未公開はネットに出さない
- **収益化＝リーチ** — デスクトップのインストール障壁を避け、ゼロインストールの Web で母数を取る
- **創作の幅** — EPUB 出力・外部サイト書き出しで、他サイトの既存読者基盤にも乗れる「道具」にする
- **発見の哲学** — PV/人気順を中心に置かない（多数派バイアスの固定化を断つ）。発見は PF 側で「本屋でぶらぶら」体験として実装（P1+）

## 3. 設計哲学（全体を貫く原則）

- **依存最小・コピーして所有**（shadcn/ui・nuqs のような「所有できEOLに縛られない」ものを優先）
- **TDD**・**構造のルール化**で保守性を担保
- **未公開はローカルのみ**（サーバに送らない）。バックアップ/端末移行＝**optin クラウド同期（at-rest 暗号化・復旧可）**＝収益化接点
- **正本フォーマットはライブラリ非依存**（エディタを替えてもデータ・変換は不変）

## 4. 新 MVP（一行で）

**「書く＋出す」** ＝ ローカルで執筆（ルビ/傍点/シーン区切り）し、EPUB / なろう / カクヨムへ書き出せる、サーバ最小のツール。詳細は [01-mvp-scope.md](./01-mvp-scope.md)。

# 01 — MVP スコープ（書く＋出す）

## 0. MVP の第一目標

ローカルで快適に**書けて**、EPUB／外部サイト記法で**持ち出せる**こと。**サーバをほぼ必要としない**（認証・クラウド同期は optin、未公開はローカルのみ）。狙いは「まず使われる」——なろう/カクヨムの既存読者基盤に乗る。

> 旧 P0（読書プラットフォームで読む/書く/読了する）とは**逆転**。読書/発見/支援/辞書は P1+。

## 1. データ構造

- **作品（連載）＞ 話（エピソード）＞ block 列**
- 1作品＝複数話を 1冊（EPUB）／連載（外部サイト）として扱える
- 本文は block 列の JSON（正本）。詳細は [02-notation-and-format.md](./02-notation-and-format.md)

## 2. 書く（執筆）

- **エディタ方式 = A1（記法方式）**: 素の `<textarea>`（shadcn）＋ 自前の記法パーサ → 正本 block JSON ＋ **ライブプレビュー**（分割表示）
  - WYSIWYG（contenteditable）は IME 問題ゆえ**不採用**
- 入力できる表現: **ルビ・傍点・シーン区切り**（記法は 02 参照）
- **最小の自動版管理**（履歴スナップショット＋手動復元。安全網。フル版管理は後段）

## 3. 出す（書き出し）

- **EPUB / 電子書籍**（**縦書き対応**）
- **小説家になろう 記法**書き出し
- **カクヨム 記法**書き出し
- 書き出しは正本 block 列を各ターゲットへ変換（記法はサイト互換なので恒等に近い）
- **手動エクスポート/インポート**（構造化バンドル）＝ 自分のデータを取り出せる escape hatch
- **フォルダ往復（話ごとのなろう記法テキスト）**: ローカルフォルダへ話単位で書き出し→取り込み（File System Access API、非対応ブラウザは DL/アップロード）。外部サイト入稿に加え、**外部 Claude（Claude Code）で編集する橋**を兼ねる

## 3.1 AI 編集支援（方針・P1 以降）

- AI はツールに**組み込まない**。**外部 Claude（Claude Code/Desktop）に委ねる**（本文はユーザーが送る先だけに出る＝漏洩防止に忠実）
- Web 版の橋 = 上記**フォルダ往復**（export → Claude Code 編集 → import 再パース、ロスレス）
- MCP は 2 形態：**read-only リモート MCP**（同期インフラ上の認証付きエンドポイント・有料クラウド束・同期と同時期に提供可）と、**ローカル stdio MCP**（Tauri デスクトップ版で Rust 側が提供・後段）。詳細は [05-sync.md §1.2](./05-sync.md)
- 我々のサーバ経由の**マネージド AI（自前推論）は採らない**（漏洩防止・サーバ最小・資金的制約）。推論は常にユーザーの AI が行う

## 4. 保存・バックアップ

- 既定 = **ローカル保存**（ネイティブ IndexedDB／PWA、`navigator.storage.persist()` で消失耐性）
- バックアップ/端末移行 = **optin アカウント＋クラウド同期（at-rest 暗号化・復旧可）**＝収益化接点
  - ※ 同期サーバの実装は MVP 内で最小か、MVP 直後か要検討（→ 99）

## 5. MVP に含めないもの（明示）

| 機能 | 行き先 |
|---|---|
| @参照 / オブジェクト辞書 / 図鑑 / 段階開示 | P1（詳細は [04-glossary.md](./04-glossary.md)。図鑑＝P1.1・段階開示＝P1+） |
| 読書体験 / 読了 / 反応ゲート / 通知 | 公開プラットフォーム（P1+） |
| 発見（棚・シャッフル）/ ミニブログ / ダッシュボード | 公開プラットフォーム（P1+） |
| 支援 / サブスク / 配分 | 公開プラットフォーム（P3/P4 相当・後段） |
| フル版管理（差分・版コメント等） | 後段 |

## 6. 受け入れ条件（ドラフト）

- [ ] テキストを書いて保存→再読込で復元できる
- [ ] ルビ `｜親文字《よみ》`・傍点・シーン区切りが正本へ正しくパースされる
- [ ] ライブプレビューが入力に追従して正しく描画する
- [ ] 作品（複数話）を EPUB（縦書き）に書き出せる
- [ ] なろう / カクヨム 記法に書き出せる（恒等に近い）
- [ ] 構造化バンドルをエクスポート→別端末でインポートして同一状態に復元できる
